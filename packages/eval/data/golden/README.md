# Golden eval dataset

Lawyer-reviewed reference contracts paired with expected outputs. The thing Parasol's eval suite measures against.

## Sprint 1 dataset (DEF-027)

20 NDA contracts representative of the Kenyan SME / mid-corporate market:
- 8 unilateral NDA (disclosing party = Parasol customer)
- 6 unilateral NDA (receiving party = Parasol customer)
- 4 mutual NDA
- 2 employee/contractor NDA

Each anonymised: party names, dates, and other PII redacted. Original commercial intent preserved. Counsel sign-off on the redaction plus the golden output before merge.

## Per-case structure

```
nda/
└── <case-id>/
    ├── source.docx         # Original contract (anonymised)
    ├── source.txt          # Plain-text extract (for eval runner)
    ├── golden.yaml         # Expected output
    ├── playbook-version    # Playbook semver this golden is calibrated against
    └── notes.md            # Lawyer notes — why this case, what it tests
```

## golden.yaml shape

```yaml
case_id: nda-unilateral-disclosing-001
contract_type: nda
nda_subtype: unilateral_disclosing
playbook_version: 0.1.0
expected_clauses:
  - clause_id: nda.confidentiality_term
    severity: medium
    found_in_source: true
    expected_redline: "Extend term from 2 years to 5 years from disclosure"
    citations_required:
      - source: kenya-law/case/2019-eklr-1234
      - source: parasol-playbook/nda#confidentiality_term
  # ... more clauses
expected_overall_severity: medium
expected_critical_count: 0
expected_high_count: 1
notes: |
  Standard unilateral NDA from a Kenyan bank's vendor onboarding. Tests
  short-term confidentiality (red flag), missing IP carve-out (high), and
  Kenyan jurisdiction clause (acceptable, no flag).
```

## Calibration discipline

- **Lawyer-rated, not Claude-rated.** Goldens are authored by Kenyan-qualified counsel, not generated by an earlier Claude run. Pairs of lawyers cross-check (DEF-029 for inter-rater calibration).
- **Versioned with playbooks.** Goldens specify which playbook version they're calibrated against. Playbook bump → goldens reviewed for impact.
- **No synthetic contracts in v1.** Real anonymised contracts only. Synthetic ones go in a separate `data/stress/` set later and don't gate releases.

## Confidentiality

Goldens are confidential. The original contracts came from Parasol's network of counsel and customer pilots under explicit anonymisation + permission-to-use terms. Even after anonymisation, this directory is not redistributed beyond the engineering team. Repository access already gates this.
